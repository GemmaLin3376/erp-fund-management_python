# ERP资金管理系统 需求文档（最终版）

> 版本：v1.0（已交付实现版）  
> 项目：arPython  
> 技术栈：Python / Flask / SQLite / Bootstrap

---

## 一、系统概述

基础ERP资金管理系统，覆盖资金收支的基础资料维护和应收应付单据管理，支持单据完整生命周期操作（新增、编辑、复制、审核、反审核、删除）及批量操作，并实现跨单据的金额自动反写联动。

---

## 二、前端布局规范

### 2.1 导航结构

采用**左侧竖排布局**，一级菜单分为两大分类：

```
├── 基础资料
│   ├── 客户管理
│   ├── 收入类别管理
│   └── 收款账户管理
└── 应收应付

首页展示**三步业务流程图**，引导用户按顺序操作：

```
┌─────────────┐      ┌─────────────┐      ┌─────────────┐
│  📋 第一步   │  ──▶ │  📄 第二步   │  ──▶ │  💰 第三步   │

## 五、核心业务规则

### 5.1 金额反写规则

收款单的**保存（新增/编辑）**和**删除**操作，必须实时更新关联的其他收入单的以下字段：

- `已收款金额`：所有关联该收入单的收款单分录金额之和（含所有状态）
- `未收款金额`：单据总金额 - 已收款金额
- `收款状态`：根据下方收款状态判定规则自动计算

> **重要**：收款单的**审核/反审核**操作**不触发**其他收入单的状态更新，仅变更收款单自身状态。

### 5.2 收款状态判定规则

```
已收款金额 >= 单据总金额  →  全部收款
已收款金额 > 0           →  部分收款
已收款金额 = 0           →  未收款（仅适用于已审核收入单）
单据未审核               →  显示"—"（灰色）
```

### 5.3 编辑收款单的金额处理规则

编辑收款单时，必须执行以下两步：
1. **先回退**：将旧分录关联收入单的已收款金额减去旧分录金额
2. **再累加**：将新分录金额累加到关联收入单的已收款金额

两步缺一不可，确保已收款金额始终等于新金额，而非旧金额+新金额的叠加。

### 5.4 编码自动生成规则

| 模块 | 前缀 | 格式示例 |
|------|------|---------|
| 客户 | `KH` | `KH001`、`KH002` |
| 收入类别 | `SR` | `SR001`、`SR002` |
| 收款账户 | `ZH` | `ZH001`、`ZH002` |
| 其他收入单 | `QTSR` | `QTSR001`、`QTSR002` |
| 收款单 | `SK` | `SK001`、`SK002` |

### 5.5 金额合法性校验规则

| 场景 | 校验规则 | 提示信息 |
|------|---------|---------|
| 分录金额未填写 | `amount is None` | 分录信息不完整 |
| 分录金额 ≤ 0 | `amount <= 0` | 金额必须大于0 |
| 收款金额超过未收款金额 | `amount > unreceived_amount` | 收款金额不能超过未收款金额 |

> **注意**：不可使用 `if not amount` 作为判断，因为 Python 中 `not 0 == True`，会将合法的 `0` 误判为"未填写"。

### 5.6 基础资料引用保护规则

| 基础资料 | 引用检查 | 被引用时操作 |
|---------|---------|------------|
| 客户 | 被其他收入单或收款单引用 | 禁止删除 |
| 收入类别 | 被其他收入单分录引用 | 禁止删除 |
| 收款账户 | 被收款单引用 | 禁止删除 |

### 5.7 单据状态转换规则

**其他收入单**：

```
未审核 --[审核]--> 已审核 --[反审核（无关联收款单）]--> 未审核
```

**收款单**：

```
未审核 --[审核]--> 已审核 --[反审核]--> 未审核
```

## 八、批量操作规范

### 8.1 其他收入单批量操作

| 操作 | 说明 | 失败原因 |
|------|------|---------|
| 批量审核 | 仅对未审核单据生效 | `{单号}：已审核单据不能再次审核` |
| 批量反审核 | 仅对**已审核**/**部分收款**/**全部收款**状态且无收款单关联的单据生效；有关联收款单时统一提示"已关联收款单"；**未审核**状态不能反审核 | `{单号}：该单据已关联收款单，不能反审核` / `{单号}：当前状态为未审核，不能反审核` |
| 批量删除 | **仅对未审核单据生效，已审核单据不能删除** | `{单号}：已审核单据不能删除` |
| 批量复制 | **不支持** | - |

### 8.2 收款单批量操作

| 操作 | 说明 | 失败原因 |
|------|------|---------|
| 批量审核 | 仅对未审核单据生效 | `{单号}：已审核单据不能再次审核` |
| 批量反审核 | 仅对已审核单据生效 | `{单号}：未审核单据不能反审核` |
| 批量删除 | 仅对未审核单据生效，**必须调用 `order.delete()` 方法**（确保关联收入单金额回退） | `{单号}：已审核单据不能删除` |

### 8.3 批量操作通用规范

- **前端行为**：
  - 确认框只显示选中总数：`"确定要删除选中的 3 条记录吗？"`
  - 提交所有选中的记录 ID 到后端（不过滤）
  
- **后端行为**：
  - 按状态过滤执行操作
  - 返回消息格式：`"操作完成：成功 N 条，失败 M 条"`（成功+失败=已选总数）
  - 部分成功时仍返回 `success: true`
  
- **失败提示**：
  - 失败消息需包含单号及具体原因，格式为 `{单号}：{失败原因}`
  - 所有批量操作（审核/反审核/删除）均支持失败详情展示

### 8.4 列表复选框交互规范

列表页有两个全选入口，行为完全一致，且互相同步：
- **批量操作栏"全选"复选框**（文字左侧）
- **表头列复选框**（数据表格第一列表头）

**行复选框点击**：
- 切换当前行选中状态（高亮/取消高亮）
- 同步更新**两个**全选框状态（所有行选中时均勾选，否则均取消）
- 更新已选数量显示

**任一全选框点击**：
- 另一个全选框立即同步为相同状态
- 所有行复选框及高亮状态同步更新
- 更新已选数量显示

### 8.5 列表分页规范

**分页控件位置**：数据表格下方，与批量操作栏垂直排列

**每页条数切换**：
- 提供下拉选择：5条、10条、20条、50条
- 默认每页显示5条
- 切换后自动回到第1页

**分页按钮**：
- 上一页、下一页、页码跳转
- 当前页码高亮显示
-  ellipsis(...)表示省略页码

**分页信息展示**：
- 格式：共 {总条数} 条，第 {当前页}/{总页数} 页

### 8.6 跨页全选规范

**全选行为**：
- 点击"全选"复选框后，选中**所有页**符合当前筛选条件的数据
- 已选数量显示为总条数（跨所有页）
- 批量操作对所有选中数据生效（含其他页）

**跨页状态持久化**：
- 使用 sessionStorage 保存跨页全选状态
- 切换到任意页面时，自动恢复当前页的选中状态（行复选框自动勾选）
- 刷新页面后跨页全选状态保持

**取消全选**：
- 取消"全选"复选框后，清除所有页的选中状态
- 清除 sessionStorage 中的持久化数据

**行复选框点击**：
- 跨页全选状态下，点击任意行复选框退出跨页全选模式
- 仅保留当前页已勾选的行
- 清除 sessionStorage 中的持久化数据
